## How it works

Securely connect your GitLab environment so teams can triage issues, review merge requests, and automate repo workflows faster, with less context switching and fewer manual updates.

### Speed up issue triage and incident response

Ask for open P0 bugs by label, summarize root causes, and create follow-up issues with owners, reducing handoffs and time-to-resolution.

### Ship cleaner code with faster MR reviews

Pull diffs, commits, and pipeline status for a merge request, then draft review comments and release notes to reduce review cycles and missed regressions.

### Find answers in code and change history

Run semantic code search to locate patterns like authentication handling, trace related issues and MRs, and share a concise summary for onboarding or audits.

## Sample prompts

Triage critical issues and recommend actions

**@GitLab Issues** Show me all open P0 issues for the last 14 days grouped by label and owner with a suggested triage order.

Create issues from an incident

**@GitLab Issues** Turn the most recent incident issue into follow up issues with clear acceptance criteria and suggested owners.

Draft release notes for recent changes

**@GitLab Issues** Generate release notes from all merge requests merged into the current milestone and group them by feature area.
